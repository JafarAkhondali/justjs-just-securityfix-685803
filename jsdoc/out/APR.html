(static) access(objectnon-null, pathopt, handleropt)

Clones `object` and accesses to a deep property in that object.
Parameters:
Name Type Attributes Default Description
object Object The base object.
path Array <optional>
 [path] The ordered keys.
handler APR~access_handler <optional>
 A custom function.
Source:
  • APR.js, line 615
Throws:
If some property causes access problems.
Type
TypeError
Returns:
If `handler` is given: the returned value of that function, otherwise: the last value of `path` in the cloned object.
Examples

Accessing to some existent property

access({a: {b: {c: {d: 4}}}}, ['a', 'b', 'c', 'd'], function (currentObject, currentKey, propertyExists, path) {
    return propertyExists ? currentObject[currentKey] : null;
}); // returns 4.

Accessing to some existent property with a non-JSON-like-object as a value

access({a: 1}, ['a', 'b', 'c']); // throws TypeError.

Accessing to some non-existent property

var obj = {z: 1, prototype: [...]};

access(obj, 'a.b.c'.split('.'), function (currentObject, currentKey, propertyExists, clonedObject, path) {
    
    if (!propertyExists) {
        clonedObject[currentKey] = path.length;
    }
    
    // At this point:
    //     `obj` stills being {z: 1, prototype: [...]},
    //     `clonedObject` has a value in `currentKey`,
    //     and `this` has all the added keys (even the ones modified in `clonedObject`).
    // so, you can return `this` to assign it to `obj` later.
    return this;

}); // returns {z: 1, a: {b: {c: 3}}, prototype: [...]}

(static) get(value, defaultValue)

Checks if `value` is `defaultValue` and returns `value` (if it's) or `defaultValue` (if it isn't).
Parameters:
Name Type Description
value * A value to test against `defaultValue`.
defaultValue APR~is_type A defaultValue value with an expected APR~is_type for `value`.
Source:
  • APR.js, line 276
Returns:
if `value` looks like `defaultValue` it returns `value`, otherwise it returns `defaultValue`.
Examples

Using a value that "looks like" `defaultValue` is the recommended usage:

get('a', ['b']); // ['b']

Note that you can use any of APR~is_type, like 'object'.

get(['a'], 'object'); // Object(['a']); -> Array['a'];

If you pass a Constructor, it will be initialized.

get(null, Function); // new Function(null); -> function anonymous();
// Please note that the Function constructor (the one used when `defaultValue` is 'function' or Function)
// may throw a CSP Error, so it's not recommended to use it. Use: function someFn () { [...] }; instead. i.e.:
// get(null, function isNotAFn() { [...] });

Source:

Returns:

Type

Examples
is(123, 4); // true: "4" is a number.

Note that the following is not the same:

is({}, 'object') === is({}, Object); // false.

Multiple arguments will be evaluated as "OR".

is(new Number(NaN), new Number(NaN), NaN, Number, 'number', Object, {}, {a: 1}); // true (in each case).
is(new Number(NaN), 'object', null, [], 123, /^NaN$/, 'NaN'); // false (in each case).

(static) parseUrl(urlopt) → {APR~urlParts}

Parses `url` without checking if it's a valid url. Note that this function uses `window.location` to make relative urls, so weird values in there will give weird results.
Parameters:
Name Type Attributes Default Description
url string <optional>
 window.location.href A relative, an absolute or a blob url.
Source:
  • APR.js, line 389
Returns:
Type
APR~urlParts
Examples

An absolute url:

parseUrl(window.location.href);

A relative url:

parseUrl('/?a#c?d'); // "/" is the pathname, "?a" the search and "#c?d" the hash.

A blob url:

parseUrl('blob:'); // Same as 'blob:' + `window.location.href`

Some good-to-know urls:

parseUrl(); // Same as `window.location`.
parseUrl('a'); // Something that doesn't start with "/", "?", or "#" is evaluated as a host.
parseUrl('a:b'); // "a:b" is a host, since "b" is not a number.
parseUrl('//'); // evals as the current origin.
parseUrl('blob://'); // Same as 'blob:' + `window.location.origin`.
